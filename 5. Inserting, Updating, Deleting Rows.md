## 1. What is a schema?

- A schema defines the structure of each table: column names, datatypes, defaults, constraints (PRIMARY KEY, NOT NULL, UNIQUE, FOREIGN KEY, etc.).    
- Schemas enforce consistency and let databases scale to millions of rows reliably.

**Example (conceptual)**  
`Movies( MovieID INT, Title VARCHAR, Year INT, Rating FLOAT )` — Year must be an integer, Title a string, etc.

# 2. Inserting rows (INSERT)

## Basic forms

**Insert values for every column (order must match table definition)**

```sql
INSERT INTO mytable
VALUES (val1, val2, val3),
       (val4, val5, val6);
```

**Insert specifying columns (recommended — forward compatible)**

```sql
INSERT INTO employees (employee_id, name, salary)
VALUES (1, 'Asha Patel', 60000),
       (2, 'Rahul Sharma', 45000);
```

- Columns listed determine the order and number of values per row.
- If new columns are later added with defaults, existing INSERTs that name columns usually keep working.

## Insert with expressions

You can use arithmetic, string functions, or other expressions in VALUES:

```sql
INSERT INTO boxoffice (movie_id, rating, sales_in_millions)
VALUES (1, 9.9, 283742034 / 1000000.0);
```

Or compute strings:

```sql
INSERT INTO users (username, email)
VALUES (LOWER('John Doe'), CONCAT('john.doe','@example.com'));
```

## Insert from another table

Useful to copy or transform rows:

```sql
INSERT INTO archived_employees (employee_id, name, salary, left_date)
SELECT employee_id, name, salary, CURRENT_DATE
FROM employees
WHERE status = 'terminated';
```

## NULL and DEFAULT

- Omitting a column that has a DEFAULT uses the default value.    
- To explicitly use default in VALUES you can (in many dialects) use `DEFAULT` keyword.

# 3. Updating rows (UPDATE)

## Syntax

```sql
UPDATE tablename
SET column1 = expr1,
    column2 = expr2
WHERE condition;
```

- The `WHERE` decides which rows change. Without it, every row is updated.
    

## Interview example: Give a 50% increment to people with salary less than a threshold

Assume `employees(employee_id, name, salary)`.

**Simple 50% raise for salary < 50,000**

```sql
UPDATE employees
SET salary = salary * 1.5
WHERE salary < 50000;
```

- This multiplies current salary by 1.5, i.e., a 50% increase.    
- If you want to increase by salary + 50% (same math) you could use: `SET salary = salary + (salary * 0.5)`    

**If you want to apply the raise but cap the new salary at e.g. 60,000**

```sql
UPDATE employees
SET salary = LEAST(salary * 1.5, 60000)
WHERE salary < 50000;
```

`LEAST` is available in many engines (Postgres, MySQL). In SQL Server use `CASE`:

```sql
UPDATE employees
SET salary = CASE
               WHEN salary * 1.5 > 60000 THEN 60000
               ELSE salary * 1.5
             END
WHERE salary < 50000;
```

## Conditional updates using CASE

Apply different increases by band:

```sql
UPDATE employees
SET salary = CASE
               WHEN salary < 30000 THEN salary * 1.6  -- 60% raise
               WHEN salary < 50000 THEN salary * 1.5  -- 50% raise
               ELSE salary
             END
WHERE salary < 50000;
```

## Update using values from another table (JOIN)

```sql
UPDATE employees e
SET salary = e.salary + a.increase_amount
FROM adjustments a
WHERE e.employee_id = a.employee_id;
```

(Exact syntax varies by database; Postgres supports this `FROM` form. MySQL uses multi-table `UPDATE` with `JOIN`.)

## Safety tips for UPDATE

1. Always test the `WHERE` with `SELECT` first:
```sql
SELECT * FROM employees WHERE salary < 50000;
```
2. Wrap changes in a transaction when possible:
```sql
BEGIN;
UPDATE ...;
-- inspect results with SELECT
COMMIT;
-- or ROLLBACK; if something looks wrong
```
3. Avoid leaving out `WHERE` unless you truly want to update all rows.
4. Consider adding an audit/log table or using `RETURNING` (Postgres) to see changed rows:
```sql
-- Postgres
UPDATE employees
SET salary = salary * 1.5
WHERE salary < 50000
RETURNING employee_id, name, salary;
```

# 4. Deleting rows (DELETE)

## Syntax

```sql
DELETE FROM mytable
WHERE condition;
```

- Without `WHERE` all rows in the table are removed.

## Examples

**Delete terminated employees**

```sql
DELETE FROM employees
WHERE status = 'terminated';
```

**Delete a single row (if you have an id)**

```sql
DELETE FROM employees
WHERE employee_id = 123;
```

## Safety tips for DELETE

1. Test the condition with `SELECT` first:
```sql
SELECT * FROM employees WHERE status = 'terminated';
```
2. Use transactions for safety:
```sql
BEGIN;
DELETE FROM employees WHERE ...;
-- verify
COMMIT;
```
3. Consider soft deletes instead of physical deletes:
    - Add column `is_deleted BOOLEAN DEFAULT FALSE` and `deleted_at TIMESTAMP`.        
    - Use `UPDATE employees SET is_deleted = TRUE, deleted_at = NOW() WHERE ...;`      
    - This preserves history and prevents accidental loss.        
4. Some DBs support `RETURNING` to show deleted rows (Postgres):
```sql
DELETE FROM employees
WHERE status = 'terminated'
RETURNING employee_id, name;
```

# 6. Short worked example (employees table)

Schema:

```sql
CREATE TABLE employees (
  employee_id INT PRIMARY KEY,
  name VARCHAR(100),
  salary NUMERIC(10,2),
  status VARCHAR(20) DEFAULT 'active'
);
```

**Insert multiple rows**

```sql
INSERT INTO employees (employee_id, name, salary)
VALUES (1, 'Aman Pawar', 42000),
       (2, 'Meera Singh', 52000),
       (3, 'Arjun Rao', 28000);
```

**Give 50% increment to employees with salary < 50000**

```sql
-- test first
SELECT employee_id, name, salary FROM employees WHERE salary < 50000;

-- then update inside a transaction
BEGIN;
UPDATE employees
SET salary = salary * 1.5
WHERE salary < 50000;
-- verify
SELECT employee_id, name, salary FROM employees WHERE employee_id IN (1,3);
COMMIT;
```

**Delete someone by id**

```sql
DELETE FROM employees WHERE employee_id = 2;
```
